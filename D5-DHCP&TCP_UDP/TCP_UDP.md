OSI layer 4 的協定，用來做傳輸資料的，IP本身只做定址，也就是知道門牌號，TCP UDP則是用來傳送資料的

UDP
->connectionless, 不會建立連線也不會ack packet有沒有正確被收到，只會一直丟，適合用在高連續性的場景例如說串流或是遊戲

TCP
->connection-oriented ,要先建立連線再傳封包，並且會有Ack封包被收到的機制
檢查的方式是封包會有有序的編號，這樣就可以知道有沒有缺或是重複，並且接收者也會傳Ack用的號碼確認有收到

三次握手:TCP建立連線的方式，由兩個flag做處理

客戶端                                            伺服器
  |                                                             |
  |--- SYN, seq=X -------------->     |
  |                                                             |
  |<-- SYN-ACK, seq=Y, ack=X+1--|
  |                                                             |
  |--- ACK, ack=Y+1 ------------>    |
1.客戶端送一個sync跟一個隨機的數字X來做確認
2.server回傳一個sync-ack並且回傳x+1表示有收到跟一個y做確認
3.client回傳一個ack跟y+1表示收到了

四次揮手:TCP表示斷線的方式，用FIN跟ACK兩個flag
客戶端                    伺服器
  |                                             |
  |-------- FIN ----------->|  「我傳完了，想關閉」
  |                                            |
  |<-------- ACK -----------|  「好，我知道你要關了」
  |                                              |
  |<-------- FIN -----------|  「我這邊也傳完了，我也要關」
  |                                             |
  |--------- ACK ---------->|  「好，確認關閉」
  |                                             |
  |===== 連線關閉==== |
