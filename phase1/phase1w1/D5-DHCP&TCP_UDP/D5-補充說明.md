# D5 補充說明

## 1. Port 號碼（Port Numbers）

TCP/UDP 用 IP 找到主機，再用 **Port 號碼**找到主機上正確的應用程式。Port 範圍 0–65535：

| 範圍 | 名稱 | 說明 |
|------|------|------|
| 0 – 1023 | Well-Known Ports | 標準服務使用，需要 root 權限才能綁定 |
| 1024 – 49151 | Registered Ports | 廠商或應用程式向 IANA 登記使用 |
| 49152 – 65535 | Dynamic / Ephemeral Ports | 客戶端連線時由 OS 隨機選用的來源 Port |

### 常見 Well-Known Ports

| Port | 協定 | 服務 |
|------|------|------|
| 20 / 21 | TCP | FTP（資料 / 控制） |
| 22 | TCP | SSH |
| 23 | TCP | Telnet（明文，不安全） |
| 25 | TCP | SMTP（Email 傳送） |
| 53 | UDP / TCP | DNS |
| 67 / 68 | UDP | DHCP（Server / Client） |
| 80 | TCP | HTTP |
| 110 | TCP | POP3（Email 收信） |
| 143 | TCP | IMAP（Email 收信） |
| 443 | TCP | HTTPS |
| 3389 | TCP | RDP（Windows 遠端桌面） |

## 2. TCP Flow Control（流量控制）

TCP 用**滑動視窗（Sliding Window）**機制避免發送方速度超過接收方的處理能力：

- 接收方在 ACK 中回報自己的**接收視窗大小（Window Size）**
- 發送方最多只能送出視窗大小的資料量，等收到 ACK 才繼續送
- 若接收方緩衝區快滿，視窗大小縮小；若緩衝區清空，視窗大小增大

```
發送方                          接收方
  |                               |
  |--- 封包 1 ─────────────────>  | 視窗 = 3
  |--- 封包 2 ─────────────────>  |
  |--- 封包 3 ─────────────────>  |
  |  (等待 ACK，不能超過視窗大小)   |
  |<── ACK 1，視窗 = 2 ─────────  |
  |--- 封包 4 ─────────────────>  |
  ...
```

## 3. DHCP 安全威脅

DHCP 本身設計沒有認證機制，容易遭受以下攻擊：

### DHCP Starvation（耗盡攻擊）
攻擊者大量偽造不同 MAC Address 的 Discover 請求，耗盡 DHCP Scope 內的所有 IP，導致正常用戶無法取得 IP（DoS）。

### Rogue DHCP Server（惡意 DHCP Server）
攻擊者在網段內架設惡意 DHCP Server，搶先回應 Offer，將受害者的 Default Gateway 或 DNS Server 指向攻擊者的主機，進行中間人攻擊（MITM）。

```
正常用戶  ──Discover──>  [惡意 DHCP 搶先回應 Offer]
                         Gateway = 攻擊者 IP
                         DNS    = 攻擊者 IP
```

### 防禦：DHCP Snooping

交換器上的安全功能，將 Port 分為兩類：

| Port 類型 | 說明 |
|-----------|------|
| Trusted Port | 連接合法 DHCP Server 的 Port，允許 DHCP Offer / ACK |
| Untrusted Port | 連接一般用戶的 Port，只允許 DHCP Discover / Request，丟棄任何 Offer / ACK |

> DHCP Snooping 可同時防禦 Starvation 和 Rogue DHCP Server。
