# D1 補充說明

## 1. PDU（Protocol Data Unit）各層資料單位

OSI 每一層對資料的稱呼不同，理解這點有助於看懂封包分析工具的輸出：

| 層號 | 名稱 | PDU 名稱 |
|------|------|---------|
| L7 應用層 | Application | Data |
| L6 呈現層 | Presentation | Data |
| L5 會話層 | Session | Data |
| L4 傳輸層 | Transport | Segment（TCP） / Datagram（UDP） |
| L3 網路層 | Network | Packet |
| L2 資料連結層 | Data Link | Frame |
| L1 實體層 | Physical | Bit |

## 2. 封裝（Encapsulation）與解封裝（Decapsulation）

資料在傳送端從 L7 往下走到 L1，每經過一層就會被加上該層的 **header**（有時還有 trailer），這個過程叫做**封裝**。

```
L7  [  Data  ]
L4  [ TCP Header | Data ]
L3  [ IP Header | TCP Header | Data ]
L2  [ Frame Header | IP Header | TCP Header | Data | Frame Trailer ]
L1  → 轉成 bits 發送
```

接收端則相反，從 L1 往上，每層剝掉自己的 header，稱為**解封裝**。

> 這就是為什麼 Wireshark 可以分層顯示各個 header 的內容。

## 3. 各層常見協定補充

| 層號 | 常見協定 |
|------|---------|
| L7 應用層 | HTTP、HTTPS、FTP、SMTP、DNS、SSH |
| L4 傳輸層 | TCP、UDP |
| L3 網路層 | IP、ICMP、OSPF、BGP |
| L2 資料連結層 | Ethernet、ARP、PPP |
| L1 實體層 | 網路線、Wi-Fi 無線電、光纖 |
