# D4 補充說明

## 1. 錯誤更正：CNAME 的正確說明

原筆記寫「一個 IP 對應多個 Domain Name（別名）」，這是不正確的。

**CNAME（Canonical Name Record）** 是讓一個 Domain Name 成為另一個 Domain Name 的別名，兩者都是名稱，與 IP 無直接關係：

```
www.example.com  CNAME  example.com    ← www 是 alias，指向 canonical name
shop.example.com CNAME  example.com    ← shop 也是 alias，指向同一個 canonical name
example.com      A      93.184.216.34  ← 最終由 A record 解析出 IP
```

查詢 `www.example.com` 時，DNS 會先查到 CNAME 指向 `example.com`，再查 `example.com` 的 A record 拿到 IP。

> CNAME 不能直接指向 IP，也不能用在根域名（apex domain）上。

## 2. DNS 使用的 Port

| 用途 | Port | 協定 |
|------|------|------|
| 一般 DNS 查詢 | 53 | UDP（小於 512 bytes 的回應） |
| 大型回應 / Zone Transfer | 53 | TCP |
| DNS over TLS | 853 | TCP |
| DNS over HTTPS | 443 | TCP |

> DNS 預設用 UDP/53 是因為速度快、開銷小。當回應資料超過 512 bytes（或 EDNS 上限），才改用 TCP。

## 3. Zone Transfer

Primary DNS Server 將整份 DNS Zone 記錄同步給 Secondary DNS Server 的機制：

- 使用 TCP/53
- 指令：`dig AXFR @dns-server domain-name`
- **安全風險**：若 Zone Transfer 未限制來源 IP，攻擊者可以拿到整個域名的所有記錄（子域名列表、MX、SPF 等），等同洩漏內部網路地圖

## 4. DNS Cache Poisoning（DNS 快取投毒）

攻擊者偽造 DNS 回應，將惡意 IP 注入受害者的 DNS Server Cache，讓查詢合法網域的用戶被導向惡意網站。

```
攻擊者         DNS Resolver          合法 DNS Server
   |                |                       |
   |  偽造回應 ─────>|                       |
   |  （在合法回應抵達之前）                  |
   |                | ← 快取惡意 IP         |
   |                |                       |
 受害者 ──查詢──> DNS Resolver 回傳惡意 IP
```

**防禦方式**：
- **DNSSEC**：對 DNS 記錄進行數位簽章，Resolver 可驗證回應是否來自合法 Server
- **DNS over TLS / HTTPS**：加密查詢，防止中間人竄改
- 限制 Recursive Resolver 只服務內部網路
