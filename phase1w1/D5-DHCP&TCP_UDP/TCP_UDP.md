# TCP 與 UDP

OSI L4（傳輸層）的兩大協定。IP 負責定址（知道門牌號），TCP/UDP 負責實際傳送資料。

## TCP vs UDP 比較

| 特性 | TCP | UDP |
|------|-----|-----|
| 連線方式 | Connection-oriented（先建立連線） | Connectionless（直接送出） |
| 可靠性 | 有 ACK 確認、序號追蹤、重傳機制 | 無，封包送出後不確認是否到達 |
| 速度 | 較慢（有控制開銷） | 較快 |
| 適用場景 | 需要可靠傳輸：HTTP、FTP、Email | 低延遲優先：串流、線上遊戲、DNS |

## TCP 三次握手（建立連線）

```
客戶端                                    伺服器
  |                                          |
  |--- SYN, seq=X ----------------------->  |  「我想連線」
  |                                          |
  |<-- SYN-ACK, seq=Y, ack=X+1 ----------  |  「好，我準備好了」
  |                                          |
  |--- ACK, ack=Y+1 --------------------->  |  「收到，開始傳輸」
  |                                          |
```

1. 客戶端送出 `SYN` 與隨機序號 X
2. 伺服器回傳 `SYN-ACK`，確認收到（ack=X+1），並附上自己的序號 Y
3. 客戶端回傳 `ACK`，確認收到伺服器的序號（ack=Y+1）

## TCP 四次揮手（斷開連線）

```
客戶端                                    伺服器
  |                                          |
  |--- FIN --------------------------------> |  「我傳完了，想關閉」
  |                                          |
  |<-- ACK --------------------------------  |  「好，我知道你要關了」
  |                                          |
  |<-- FIN --------------------------------  |  「我這邊也傳完了，要關了」
  |                                          |
  |--- ACK --------------------------------> |  「好，確認關閉」
  |                                          |
  |=============== 連線關閉 ================ |
```

> 需要四次是因為 TCP 是全雙工（Full-duplex），雙方各自需要發送 FIN 並等待對方的 ACK。
